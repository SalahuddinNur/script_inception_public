Path management tools:
======================
Octave and Matlab:
------------------
path -> Modify or display Octave’s load path.

addpath -> Add named directories to the function search path.
rmpath -> Remove dir1, … from the current function search path.

savepath -> Save the unique portion of the current function search path that is not set during Octave’s initialization process to file.

genpath -> Return a path constructed from dir and all its subdirectories.
pathsep -> Query or set the character used to separate directories in a path.

rehash -> Reinitialize Octave’s load path directory cache. / Refresh function and file system path caches
restoredefaultpath -> Restore Octave’s path to its initial state at startup.

Octave-only:
------------
file_in_loadpath -> Return the absolute name of file if it can be found in the list of directories specified by path.
command_line_path -> Return the command line path variable.
dir_in_loadpath -> Return the full name of the path element matching dir.
pathdef -> Return the default path for Octave.

Octave save/load process:
-------------------------
savepath(file)
source(file)

file will contain something like this:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
## Begin savepath auto-created section, do not edit
addpath('dir1:dir2', '-begin');
## End savepath auto-created section

Matlab save/load process:
-------------------------
savepath(folderName/pathdef.m)
To save a search path and automatically use it in a future session, use the savepath folderName/pathdef.m syntax and set the folder specified by folderName as the MATLAB startup folder.
-> i.e. put it in userpath/pathdef.m

file will contain something like this:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
function p = pathdef
%PATHDEF Search path defaults.
%   PATHDEF returns a string that can be used as input to MATLABPATH
%   in order to set the path.

  
%   Copyright 1984-2002 The MathWorks, Inc.
%   $Revision: 1.4.2.1 $ $Date: 2003/01/16 12:51:34 $


% DO NOT MODIFY THIS FILE.  IT IS AN AUTOGENERATED FILE.  
% EDITING MAY CAUSE THE FILE TO BECOME UNREADABLE TO 
% THE PATHTOOL AND THE INSTALLER.

p = [...
%%% BEGIN ENTRIES %%%
     'H:\matlab_scripts;', ...
     matlabroot,'\toolbox\matlab\general;', ...
     matlabroot,'\toolbox\rtw\targets\xpc\xpc\xpcmngr;', ...
     matlabroot,'\work;', ...
%%% END ENTRIES %%%
     ...
];

p = [userpath,p];

Notes:
------
warning: the 'userpath' function is not yet implemented in Octave
